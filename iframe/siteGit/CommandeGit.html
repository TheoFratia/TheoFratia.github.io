Les commandes Git

git add

Cette commande met à jour l’index en utilisant le contenu actuel trouvé dans l’arbre de travail, pour préparer le contenu de la prochaine validation.



Voir plus »





git commit -m

Crée un nouveau commit contenant le contenu actuel de l’index et avec le message de validation décrivant la modification, -m est utiliser pour accompagner le commit d'un message.



Voir plus »





git status

Montre les chemins qui ont des différences entre le fichier d’index et le commit HEAD actuel, les chemins qui ont des différences entre l’arbre de travail et de fichier d’index, et les chemins dans l’arbre de travail qui ne sont pas suivis par Git .



Voir plus »





git checkout

git checkout sert a crée une nouvelle branche. Spécifier -b provoque la création d’une nouvelle branch, Pour crée une nouvelle branch il est aussi possible d'utiliser git branch nom de la branch.

git pull

git pull sert à récupérer les dossiers mis à jour dans le dépôt gitlab.

git clone

git clone sert à récupérer tous les dossiers du dépôt même ceux déjà présents.

git merge

git merge sert à fusionner deux branchs.

Comment crée un projet gitlab

  • git init
  • git remote add origin https://gitlab.com/Theo.fratia/nomdepot.git
  • git add *
  • git commit -m "message"
  • git push -u origin master
